# 接口
## 查询系统配置（已完成）
* 该接口使用Post请求
* 接口地址：`/v1/oracle/config`
### 查询参数
* configKey 配置KEY(必填,盘口背景=`oracle_market_depth_background`/盘口规则=`oracle_market_depth_rule`)
### 查询逻辑
* 先从缓存 `sys:config:${cfgGroup}:${cfgKey}` 获取
* 如果缓存没有，再从 `sys_config` 表中读取
* 数据库读取成功后，需回写缓存
* 缓存不命中且数据库无配置时，返回空字符串
### 响应字段
* data 配置内容(JSON格式字符串)

### 响应示例
```json
{
  "code": 200,
  "msg": "success",
  "data": "{\"title\":\"DOGE/USDT\",\"backgroundUrl\":\"https://xxx/banner.png\",\"subTitle\":\"热门盘口\"}"
}
```

```json
{
  "code": 200,
  "msg": "success",
  "data": "{\"title\":\"规则说明\",\"content\":[\"1. 预测结果以下一期结算价为准\",\"2. 开奖后自动结算\"]}"
}
```

# 实现口径
1. `configKey` 只允许两种值：
   * `oracle_market_depth_background`
   * `oracle_market_depth_rule`
2. 根据当前语言映射真实 `cfgGroup/cfgKey`
3. `oracle_market_depth_background` 和 `oracle_market_depth_rule` 都按多语言配置读取
4. 返回时直接透传 `cfgValue` 到 `data`，不再额外封装对象

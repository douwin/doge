# 进行中转待开奖补充任务实现方案

## 目标
* 在“预测市场期次从进行中流转为待开奖”任务中补充预创建开奖记录逻辑
* 当 `oracle_issue.status` 从 `1` 更新为 `3` 时,同步写入一条 `oracle_draw`
* 为后续“执行开奖”任务提供稳定的待开奖主记录

## 方案边界
* 本补充任务只负责预创建 `oracle_draw`
* 不在本阶段生成 `oracle_income`
* 不在本阶段修改 `oracle_order.status`
* 不在本阶段将 `oracle_issue.status` 更新为 `4`

## 设计原则
* 任务入口锁与单期次锁保持现有实现不变
* 单期次内采用事务,保证“创建开奖记录”和“更新期次状态”一致成功或一致失败
* 必须支持幂等,避免重复插入多条 `oracle_draw`
* 除入口日志外,业务处理过程需输出期次摘要、开奖记录摘要、订单数和状态流转结果日志

## 处理流程
1. 查询 `oracle_issue.status = 1` 且 `end_time < now` 的期次
2. 进入单期次处理后再次查询期次,二次校验 `status = 1`
3. 查询 `oracle_draw` 是否已存在 `issue_id = 当前期次ID` 的记录
4. 若不存在,则创建一条 `oracle_draw.status = 1` 的待开奖记录
5. 若已存在 `oracle_draw.status = 1` 或 `3`,说明该期次已预创建开奖记录,不重复插入
6. 若已存在 `oracle_draw.status = 2` 或 `4`,记录异常日志并跳过,避免覆盖后续处理结果
7. 完成上述处理后,将 `oracle_issue.status` 更新为 `3=待开奖`

## 预创建开奖记录建议字段
* `draw_no`: 生成唯一开奖记录编号
* `issue_id`
* `issue_code`
* `issue_market_config_id`
* `market_config_name`: 通过 `oracle_issue_market_config.name` 回填
* `symbol`
* `trigger_mode = 1`
* `trigger_time = 当前时间`
* `open_price = oracle_issue.open_price`
* `close_price = oracle_issue.close_price`
* `total_amount = oracle_issue.total_amount`
* `total_fee = oracle_issue.total_fee`
* `total_effective_amount = oracle_issue.total_effective_amount`
* `positive_amount = oracle_issue.positive_amount`
* `negative_amount = oracle_issue.negative_amount`
* `order_num`: 按 `oracle_order.issue_id` 统计
* `income_num = 0`
* `status = 1`
* `remark`: 可为空; 若 `close_price` 尚未准备好,可写“等待结算价”

## 幂等规则
* 同一个 `issue_id` 只允许存在一条当前有效开奖记录
* 若已存在 `status = 1` 或 `3` 的记录,本次只更新 `oracle_issue.status`,不重新插入
* 若后续需要补齐冗余字段,建议走 update 而不是 insert

## 事务建议
以下动作放在同一个事务中:
* 查询并判断 `oracle_draw` 是否已存在
* 插入或补充更新 `oracle_draw`
* 更新 `oracle_issue.status = 3`

## 对后续开奖任务的影响
* “执行开奖”任务不能再以“存在开奖记录”作为跳过条件
* 应改为:
  `oracle_draw.status = 1` 时继续开奖
* `oracle_draw.status = 3` 时补数据后继续重试开奖
* `oracle_draw.status = 2` 时跳过
* `oracle_draw.status = 4` 时按并发中的记录处理

## SQL建议
```sql
SELECT
    id,
    issue_code,
    issue_market_config_id,
    symbol,
    open_price,
    close_price,
    total_amount,
    total_fee,
    total_effective_amount,
    positive_amount,
    negative_amount,
    status
FROM
    oracle_issue
WHERE
    status = 1
    AND end_time < NOW()
ORDER BY
    end_time ASC,
    id ASC;
```

```sql
SELECT
    id,
    issue_id,
    draw_no,
    status
FROM
    oracle_draw
WHERE
    issue_id = #{issueId}
ORDER BY
    id DESC
LIMIT 1;
```

```sql
SELECT
    COUNT(1)
FROM
    oracle_order
WHERE
    issue_id = #{issueId};
```

# 接口
## 支付
* 表oracle_order
* 表account_oracle
### 接口信息
* 请求地址 `/v1/oracle/pay`
* 请求方式 `POST`
* Token `是`

### 请求参数
* id 期次ID
* choiceValue 选择值(1=UP/YES,2=DOWN/NO)
* amount 金额
* safetyPassword 资金密码(加密传输,处理方式参考MineController的buyElect)
### 响应字段
异常情况必须按照现有规范返回错误信息
* Boolean 

### 逻辑
1. 必须校验资产是否充足(可参考IMineService中的buyElect)
2. 必须判断是否允许购买多次oracle_issue_market_config.limit_mode
3. 必须加锁,同一个用户在同一时间只能支付一笔
4. 判断当前期次是否可购买,需按照oracle_issue_market_config.start_advance_minutes和需按照oracle_issue_market_config.cutoff_advance_minutes进行判断(参与表示是否能购买)
5. 支付后落库oracle_order
6. 落库后需处理资产流水(可参考IMineService中的buyElect)
7. 必须保证同一事务中处理
8. 支付成功后需同步维护account_oracle用户汇总

### 已确认口径
1. 支付币种按 `USDT` 处理，`amount` 单位为 `USDT`
2. 必须校验资金密码，处理方式参考 `MineController.buyElect`
3. `amount` 必须大于 `0`，并按 `oracle_issue_market_config.min_pay_amount`、`oracle_issue_market_config.max_pay_amount` 校验支付范围
4. 是否可购买按时间窗口判断：
   当前时间 >= `begin_time - start_advance_minutes`
   当前时间 <= `end_time - cutoff_advance_minutes`
   不满足时直接报错
5. 除时间窗口外，还需校验 `oracle_issue.status`，仅 `1=进行中`、`2=待开始` 允许支付
6. `limit_mode=1` 时，同一用户同一期次只允许支付 `1` 次；`limit_mode=2` 时允许多次支付
7. 需要新增 Oracle 专用账单和资产流水分类：
   `BillSubTypeEnum.ORACLE_PAY`
   `AssetLogSourceEnum.ORACLE_PAY`
8. 需要新增 Oracle 专用错误码，至少覆盖以下场景：
   期次不存在
   当前期次不可参与
   超出单次最小/最大金额
   当前期次不允许重复购买
   余额不足
   本次提交后的有效金额超过 `maxHoldRatio`
9. 手续费按 `oracle_issue_market_config.fee_rate` 计算：
   `fee = amount * fee_rate`
   `effective_amount = amount - fee`
10. `maxHoldRatio` 校验口径：
   按“本次提交后的有效金额”校验
   不按当前用户在整期期次中的累计占比校验
   即本次订单只基于当前提交产生的 `effective_amount` 判断是否超限
   当期次当前 `total_effective_amount <= 0` 时，本次不做 `maxHoldRatio` 拦截，避免首笔订单无法参与
11. `oracle_order` 落库字段按以下规则处理：
   `order_no` 使用现有订单号生成工具
   `account_id/account_no` 取当前登录用户
   `issue_id/issue_code/issue_market_config_id/symbol` 取当前期次
   `market_config_name` 取 `oracle_issue_market_config.name`
   `choice_value/amount/fee_rate/fee/effective_amount` 按本次支付数据写入
   `status=1(待开奖)`
12. 订单落库成功后，需要同步回写 `oracle_issue` 聚合字段，避免前面已完成的查询接口数据不一致：
   `total_amount`
   `total_fee`
   `total_effective_amount`
   `positive_num/positive_amount`
   `negative_num/negative_amount`
13. 同一个用户同一时间只能支付一笔，锁建议按用户维度处理：
   `lock:account:oracle:pay:{accountId}`
14. 资产处理按现有支付逻辑执行：
   先扣减用户 `USDT` 余额
   再写钱包账单
   整个支付、订单落库、聚合回写必须放在同一事务内
15. 支付成功后需同步维护 `account_oracle`：
   若当前用户不存在汇总记录，则新增一条
   `total_pay_amount += amount`
   `total_fee_amount += fee`
   `total_income_amount` 支付时不变，待开奖结算后再处理
   `total_profit_amount` 支付时不变，待开奖结算后再处理

# 开发
1. 数据库:mysql
2. 开发语言:java

# 要求
1. 执行“连接数据库”加载skill,通过提供的表读取表结构
2. 一定要遵循现有代码的规范
3. 关键位置和方法需要中文注释
4. SQL需要有一个很好的可读性格式
5. 写代码时一直执行，无需我确认
6. 禁止修改表结构,如果必须修改表结构只给出sql语句
7. 返回的时间、Long 、BigDecimal 都必须转化成String类型
8. 非特殊说明全部使用post请求
9. 不能修改现有业务逻辑
10. 所有编辑、启用、禁用都需要按照id进行去操作
11. 所有参数和响应字段都需要使用驼峰命名
12. 请求中除tinyint、Int、BigDecimal 外其他全部用string类型
13. 响应数据中,如果是枚举的,都需要返回name,比如marketType对应的是枚举,那么响应字段中需要增加marketTypeName字段,并根据国际化返回值

# 数据表
oracle_order

# 业务功能 
1. 列表查询(分页)
2. 统计


# 功能说明
## 列表
### 查询条件
`order_no`  '订单号',
`account_no` 'UID'
market_config_id 市场(选择框.关联表oracle_market_config),通过oracle_order.issue_market_config_id关联oracle_issue_market_config.id,参数market_config_id=oracle_issue_market_config.market_config_id
status 订单状态(选择框,关联枚举:OracleOrderStatusEnum)
### 响应字段
`id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
`order_no` varchar(64) NOT NULL COMMENT '订单号，如202605150001',
`account_id` bigint NOT NULL COMMENT '用户ID[关联表:account]',
`account_no` varchar(50) DEFAULT '' COMMENT 'UID(冗余)',
`issue_id` bigint NOT NULL COMMENT '期次ID[关联表:oracle_issue]',
`issue_code` varchar(64) DEFAULT '' COMMENT '期次编码(冗余)',
`issue_market_config_id` bigint NOT NULL COMMENT '期次市场配置ID[关联表:oracle_issue_market_config]',
`market_config_name` varchar(64) DEFAULT '' COMMENT '市场名称(冗余)',
`symbol` varchar(64) DEFAULT '' COMMENT '交易对(冗余)',
`choice_value` tinyint(1) NOT NULL COMMENT '用户选择: 1=UP/YES, 2=Down/No',
`amount` decimal(40,18) DEFAULT '0.000000000000000000' COMMENT '订单金额',
`fee_rate` decimal(20,4) DEFAULT '0.0000' COMMENT '手续费率(冗余)',
`fee` decimal(40,18) DEFAULT '0.000000000000000000' COMMENT '手续费金额',
`effective_amount` decimal(40,18) DEFAULT '0.000000000000000000' COMMENT '有效参与金额',
`draw_id` bigint DEFAULT NULL COMMENT '开奖记录ID[关联表:oracle_draw]',
`draw_no` varchar(64) DEFAULT '' COMMENT '开奖记录编号(冗余)',
`profit_result_id` bigint DEFAULT NULL COMMENT '关联收益结果ID，无外键约束',
`profit_result_no` varchar(64) DEFAULT '' COMMENT '收益编号',
`status` tinyint(1) DEFAULT '1' COMMENT '订单状态: 1=待开奖, 2=已结算-命中, 3=已结算-未命中',
`settled_at` datetime DEFAULT NULL COMMENT '结算完成时间',
`created_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
`updated_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
PRIMARY KEY (`id`)

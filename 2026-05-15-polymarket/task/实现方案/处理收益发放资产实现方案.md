# 处理收益发放资产实现方案

## 目标
* 在 `doge-task` 中新增“处理收益发放资产”定时任务
* 以 `oracle_income` 作为主驱动表发放预测市场收益
* 对 `oracle_income.bill_status = 2` 且 `status = 1` 的记录执行资产发放
* 发放成功后回写 `oracle_income.bill_status = 1`
* 生成钱包账单、资产流水，并按币种锁仓配置处理用户资产

## 方案边界
* 本任务只负责收益资产发放
* 本任务不负责生成 `oracle_income`
* 本任务不负责修改 `oracle_draw`、`oracle_issue`、`oracle_order`
* 本任务默认不维护 `account_oracle`
* `account_oracle.total_income_amount`、`total_profit_amount` 更适合在“执行开奖”生成 `oracle_income` 时维护

## 前置约束
* “执行开奖”任务已经提前生成 `oracle_income`
* 命中收益记录:
  `status = 1`, `bill_status = 2`
* 未命中收益记录:
  `status = 2`, `bill_status = 3`
* 系统资金池账户归集收益记录:
  同样按 `status = 1`, `bill_status = 2` 进入本任务发放
* 发放金额按 `income_amount` 处理，不按 `profit_amount` 过滤
* 业务日志需覆盖收益记录加载、资产币种加载、钱包账单写入、资产处理分支和账单状态回写

## 定时任务设计
### 调度时间
* `0 10 * * * ?`

### 锁设计
* 任务入口锁: `lock:handle:oracle:income:distribute`
* 单记录锁: `lock:handle:oracle:income:distribute:{incomeId}`
* 必须同时保留“任务入口锁 + 单条数据锁”,与现有 `doge-task` 规范保持一致

## 执行流程
1. 定时任务查询 `oracle_income.bill_status = 2`、`status = 1`、`income_amount > 0` 的收益记录,按 `id asc` 处理
2. 每条收益记录单独加锁后进入处理方法
3. 再次查询收益记录,二次校验 `bill_status = 2`、`status = 1`
4. 若 `income_amount <= 0`,说明当前记录不需要发放,直接跳过
5. 查询 `USDT` 资产币种配置,预测市场收益默认按 `USDT` 发放
6. 先写钱包账单,账单方向为 `ADD`
7. 若资产币种开启锁仓,则写入锁仓记录,不直接增加可用余额
8. 若资产币种未开启锁仓,则增加用户资产可用余额
9. 发放成功后将 `oracle_income.bill_status` 更新为 `1=已发放`
10. 同步更新 `oracle_income.updated_time`

## 处理口径
* 发放主条件:
  `bill_status = 2 and status = 1 and income_amount > 0`
* 发放金额:
  `income_amount`
* 未命中收益:
  `bill_status = 3`,不进入发放任务
* 已发放收益:
  `bill_status = 1`,不重复处理
* 系统资金池收益:
  与普通中奖收益使用同一套发放逻辑

## 钱包账单与资产处理
* 处理资产时必须生成钱包账单
* 处理资产时必须生成资产流水
* 处理资产时必须增加用户资产,若币种开启锁仓则改为写入锁仓记录
* 建议优先复用现有收益发放链路的处理方式:
  `assetWalletBillService.saveBill`
  `accountUserAssetService.addBalance`
  `assetLockService.saveLog`

## account_oracle 处理建议
* 本任务不建议维护 `account_oracle`
* 原因一:
  `account_oracle` 属于业务结算汇总,不是“已发放汇总”
* 原因二:
  未命中记录不会进入发放任务,若在本任务中维护 `total_profit_amount`,会遗漏亏损数据
* 原因三:
  在“执行开奖”生成 `oracle_income` 时同步维护,与结算结果一致性更好

## 异常兜底
### 异常场景
* 收益记录不存在
* 收益记录状态已变化
* `USDT` 资产币种不存在
* 钱包账单写入失败
* 资产增加失败
* 锁仓记录写入失败
* 更新 `oracle_income.bill_status` 失败

### 兜底处理
* 当前事务整体回滚
* `oracle_income.bill_status` 保持 `2=未发放`
* 下次定时任务继续重试
* 日志明确记录失败原因,便于后续排查

## 事务建议
* 以下动作必须放在同一个事务里
* 写入钱包账单
* 增加用户资产或写入锁仓记录
* 更新 `oracle_income.bill_status`

## SQL建议
```sql
SELECT
    id,
    income_no,
    order_id,
    order_no,
    draw_id,
    draw_record_no,
    account_id,
    issue_id,
    issue_code,
    income_amount,
    profit_amount,
    status,
    bill_status
FROM
    oracle_income
WHERE
    bill_status = 2
    AND status = 1
    AND income_amount > 0
ORDER BY
    id ASC;
```

```sql
SELECT
    id,
    symbol,
    lock_status
FROM
    asset_coin
WHERE
    symbol = 'USDT'
LIMIT 1;
```

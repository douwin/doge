# 查询权益详情

## 接口信息
| 项 | 内容 |
|---|---|
| 请求地址 | `/api/v1/club/member/benefit/info` |
| 请求方式 | `GET` |
| Token | 必填 |

## 业务说明
1. 根据会员权益ID返回权益详情。
2. 当前接口会根据 `benefitType` 返回不同的数据结构。

## 请求参数
| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| `benefitId` | String | 是 | 会员权益ID |

## 请求示例
```http
GET /api/v1/club/member/benefit/info?benefitId=50001
Language: zh_CN
Token: eyJhbGciOiJIUzI1NiJ9.demo
```

## 响应数据
`data` 为对象

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | String | 会员权益ID |
| `benefitType` | Integer | 权益类型 |
| `benefitTypeName` | String | 权益类型名称 |
| `elonInfo` | Object | `benefitType=1` 时返回 |
| `voucherList` | Array | `benefitType=2` 时返回；其他类型默认返回空数组 |
| `pointsInfo` | Object | `benefitType=3` 时返回 |

`elonInfo` 结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `totalNum` | String | 已发放总数 |
| `unit` | String | 单位 |

`voucherList` 元素结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | String | 票证ID |
| `voucherCode` | String | 票证码 |
| `status` | Integer | 状态：`1=未核销`，`2=已核销`，`3=已失效` |
| `statusName` | String | 状态名称 |

`pointsInfo` 结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `totalPoints` | String | 总积分 |
| `usedPoints` | String | 已使用积分 |
| `remainingPoints` | String | 剩余积分 |

## 校验说明
1. `benefitId` 不能为空。
2. `benefitId` 必须可转为数值，否则返回参数校验失败。
3. `benefitId` 对应的 `club_member_benefit` 不存在时，返回 `CLUB_MEMBER_BENEFIT_NOT_EXISTS`。
4. 会员权益存在但不属于当前登录账号时，也返回 `CLUB_MEMBER_BENEFIT_NOT_EXISTS`。

## 复杂业务说明
1. `benefitType=1` 时：
   - 优先统计 `club_order_benefit` 中当前会员、当前账号、权益类型为 ELON 且状态为“已发放”的 `config_num` 总和。
   - 若未查到已发放记录，则回退使用 `club_member_benefit.config_num`。
2. `benefitType=2` 时：
   - 数据来源于 `club_benefit_ticket`。
   - 查询条件为 `accountId + memberId + benefitType=2`。
   - 排序规则为 `status asc, createTime desc, id desc`，即未核销优先展示。
3. `benefitType=3` 时：
   - `totalPoints` 取 `club_member_benefit.config_num`。
   - `remainingPoints` 取 `club_member.points`。
   - `usedPoints = max(totalPoints - remainingPoints, 0)`。
4. 当前接口仅对权益类型 `1/2/3` 返回差异化明细；若后续存在 `4/5` 类型会员权益，当前实现仅返回基础字段，其余扩展字段为空。

## 失败码
| 错误码 | 说明 |
|---|---|
| `10001` | 参数校验失败 |
| `10056` | Club会员权益不存在 |

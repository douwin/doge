# 续费

## 接口信息
| 项 | 内容 |
|---|---|
| 请求地址 | `/api/v1/club/renew` |
| 请求方式 | `POST` |
| Token | 必填 |

## 业务说明
1. 用户对指定会员卡执行续费。
2. 成功后返回续费订单信息和续费后的会员信息。
3. 当前实现续费时同样不会写入 `club_member_benefit`、`club_benefit_ticket`、`club_points_log`，这些后续通过统一任务处理。

## 请求参数
请求体为 JSON

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| `clubConfigId` | String | 是 | Club配置ID |
| `memberId` | String | 是 | 会员ID |
| `amount` | String | 是 | 支付金额 |
| `safetyPassword` | String | 是 | 安全密码 |

## 请求示例

## 响应数据
`data` 为对象

| 字段 | 类型 | 说明 |
|---|---|---|
| `orderNo` | String | 订单号 |
| `billId` | String | 钱包账单ID |
| `renewBeforeExpireDate` | String | 续费前到期时间戳(毫秒) |
| `renewAfterExpireDate` | String | 续费后到期时间戳(毫秒) |
| `createTime` | String | 成交时间戳(毫秒) |
| `payCoinId` | String | 支付币种ID |
| `payCoinName` | String | 支付币种名称 |
| `payAmount` | String | 支付金额 |
| `memberInfo` | Object | 续费后会员信息 |

`memberInfo` 结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `memberId` | String | 会员ID |
| `memberNo` | String | 会员卡号 |
| `validStart` | String | 生效开始时间戳(毫秒) |
| `validEnd` | String | 生效结束时间戳(毫秒) |
| `status` | Integer | 会员状态 |
| `statusName` | String | 会员状态名称 |

## 校验说明
1. `clubConfigId`、`memberId`、`safetyPassword` 不能为空。
2. `amount` 不能为空，且必须大于 `0`。
3. `clubConfigId`、`memberId` 必须可转为数值，否则返回参数校验失败。
4. 当前账号存在并发中的 Club 购买/续费请求时，获取锁失败，返回 `REQUEST_TOO_OFTEN`。
5. Club 不存在时返回 `CLUB_NOT_EXISTS`。
6. Club 状态不是启用、支付币种不存在、无权益配置、年费无效、期限无效，或当前会员不满足续费条件时，统一返回 `CLUB_RENEW_NOT_ALLOWED`。
7. `memberId` 对应会员不存在、会员不属于当前账号、或会员与 `clubConfigId` 不匹配时，返回 `CLUB_MEMBER_NOT_EXISTS`。
8. 服务端年费与请求 `amount` 不一致时，返回 `CLUB_PRICE_CHANGED`。
9. 支付币种余额不足时，返回 `INSUFFICIENT_BALANCE`。
10. 安全密码错误时，返回 `SAFE_PASSWORD_ERROR`。

## 复杂业务说明
1. 并发控制：
   - 与购买共用账号级分布式锁，锁键为 `lock:account:club:trade:{accountId}`。
   - 同一账号不能同时发起续费和购买。
2. 续费金额规则：
   - 续费不走折扣，支付金额固定取 `club_config.annual_fee`。
   - `annual_fee <= 0` 或 `deadline <= 0` 时不允许续费。
3. 续费资格判断：
   - 会员状态为已停用、已失效时不允许续费。
   - `validEnd` 为空时不允许续费。
   - 当前时间未超过 `validEnd` 时，只有进入提前续费窗口才允许续费。
   - 超过 `validEnd` 但仍在宽限期内时，是否允许续费由 `grace_action` 是否为“续原卡”决定。
   - 超过宽限期后，是否允许续费由 `post_grace_action` 是否为“续原卡”决定。
4. 续费有效期规则：
   - `renewBeforeExpireDate` 取续费前会员的 `validEnd`。
   - 续费从 `max(原到期日, 当前时间)` 开始顺延 `deadline` 年。
   - `renewAfterExpireDate`、写回会员表的 `validEnd`、以及 `graceTime` 都会按“只保留年月日”入库。
   - `graceTime = renewAfterExpireDate + gracePeriodDays`。
5. 会员更新规则：
   - 续费成功后把会员状态更新为正常。
   - `validStart` 保持原值不变。
   - `accountNo` 会同步补齐为当前账号编号，来源于 `account.account_no`。
   - 若历史会员 `validStart` 为空，则按会员创建时间兜底补齐，并按“只保留年月日”写回。
   - 若原会员没有 `memberNo`，则按 `DGC + (100000 + memberId)` 自动补齐。
6. 订单和权益落库规则：
   - 新增 `club_order` 一条，订单类型为“续原卡”。
   - `club_order.order_no` 使用 `IdNoUtils.createOrderNo()` 生成。
   - `club_order_benefit.order_no` 也使用 `IdNoUtils.createOrderNo()` 逐条生成。
   - 新增 `club_order_benefit` 时，按当前 `club_config_benefit` 查询结果逐条写入，不按权益类型或状态过滤。
   - 钱包账单子类型写入 `CLUB_RENEW`。

|---|---|

| `10027` | 安全密码错误 |
| `10049` | Club不存在 |
| `10053` | Club价格已变化，请刷新后重试 |
| `10054` | Club会员不存在 |
| `10055` | 当前会员不可续费 |

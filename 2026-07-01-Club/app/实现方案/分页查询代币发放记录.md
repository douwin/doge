# 分页查询代币发放记录

## 接口信息
| 项 | 内容 |
|---|---|
| 请求地址 | `/api/v1/club/member/token/list` |
| 请求方式 | `GET` |
| Token | 必填 |

## 业务说明
1. 分页查询指定会员的代币发放记录。
2. 当前接口仅查询 `club_order_benefit` 中 `benefitType=1` 的数据。
3. 当前接口要求显式传入 `memberId`，用于区分同账号下不同会员卡。

## 请求参数
| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| `clubConfigId` | String | 是 | Club配置ID |
| `memberId` | String | 是 | 会员ID |
| `pageNo` | Integer | 否 | 页码，默认 `1`，继承自 `PageReq` |
| `pageSize` | Integer | 否 | 每页数量，默认 `10`，继承自 `PageReq` |

## 请求示例
```http
GET /api/v1/club/member/token/list?clubConfigId=10001&memberId=20001&pageNo=1&pageSize=10
Language: zh_CN
Token: eyJhbGciOiJIUzI1NiJ9.demo
```

## 响应数据
`data` 为分页对象

`list` 元素结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | String | 记录ID |
| `amount` | String | 数量，取 `club_order_benefit.config_num` |
| `createTime` | String | 创建时间戳(毫秒) |
| `status` | Integer | 状态：`1=待发放`，`2=已发放` |
| `statusName` | String | 状态名称 |

## 校验说明
1. `clubConfigId`、`memberId` 不能为空。
2. `clubConfigId`、`memberId` 必须可转为数值，否则返回参数校验失败。
3. Club 不存在时返回 `CLUB_NOT_EXISTS`。
4. `memberId` 对应会员不存在、会员不属于当前账号、或会员与 `clubConfigId` 不匹配时，返回 `CLUB_MEMBER_NOT_EXISTS`。

## 复杂业务说明
1. 数据来源于 `club_order_benefit`，查询条件为 `accountId + clubConfigId + clubMemberId + benefitType=1`。
2. 排序规则为 `createTime desc, id desc`。
3. `statusName` 使用订单权益状态术语：待发放、已发放。
4. 当没有代币发放记录时，接口仍返回标准分页结构，`list=[]`，`totalSize=0`。

## 失败码
| 错误码 | 说明 |
|---|---|
| `10001` | 参数校验失败 |
| `10049` | Club不存在 |
| `10054` | Club会员不存在 |

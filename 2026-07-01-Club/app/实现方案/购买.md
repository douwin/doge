# 购买

## 接口信息
| 项 | 内容 |
|---|---|
| 请求地址 | `/api/v1/club/purchase` |
| 请求方式 | `POST` |
| Token | 必填 |

## 业务说明
1. 用户购买新的 Club 会员卡。
2. 成功后返回购买订单号和账单ID，会员、订单、订单权益、库存和账单在服务端落库。
3. 当前实现购买时不会写入 `club_member_benefit`、`club_benefit_ticket`、`club_points_log`，这些后续通过统一任务处理。

## 请求参数
请求体为 JSON

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| `clubConfigId` | String | 是 | Club配置ID |
| `amount` | String | 是 | 支付金额 |
| `safetyPassword` | String | 是 | 安全密码 |

## 请求示例
```http
POST /api/v1/club/purchase
Language: zh_CN
Token: eyJhbGciOiJIUzI1NiJ9.demo
Content-Type: application/json

{
  "clubConfigId": "10001",
  "amount": "800",
  "safetyPassword": "123456"
}
```

## 响应数据
`data` 为对象

| 字段 | 类型 | 说明 |
|---|---|---|
| `orderNo` | String | 购买订单号 |
| `billId` | String | 钱包账单ID |

## 响应示例
```json
{
  "success": true,
  "ts": "1784592006123",
  "code": 200,
  "msg": "success",
  "data": {
    "orderNo": "2026072201234567890",
    "billId": "30001"
  }
}
```

## 校验说明
1. `clubConfigId` 不能为空。
2. `amount` 不能为空，且必须大于 `0`。
3. `safetyPassword` 不能为空。
4. 当前账号未登录或登录失效时，接口不可调用。
5. 当前账号存在并发中的 Club 购买/续费请求时，获取锁失败，返回 `REQUEST_TOO_OFTEN`。
6. `clubConfigId` 对应 Club 不存在时，返回 `CLUB_NOT_EXISTS`。
7. Club 状态不是启用、支付币种不存在、或没有任何权益配置时，返回 `CLUB_NOT_BUYABLE`。
8. 当前账号在当前规则下不允许新购时，返回 `CLUB_ALREADY_PURCHASED`。
9. Club 总库存已满时，返回 `CLUB_SOLD_OUT`。
10. 服务端按最新库存和折扣重新计算后的应付金额与请求 `amount` 不一致时，返回 `CLUB_PRICE_CHANGED`。
11. 支付币种余额不足时，返回 `INSUFFICIENT_BALANCE`。

## 复杂业务说明
1. 并发控制：
   - 控制器层按账号加分布式锁，锁键为 `lock:account:club:trade:{accountId}`。
   - 同一账号不能同时发起购买和续费。
   - 锁租期 5 分钟，获取等待时间 5 秒。
2. 库存和价格处理：
   - 事务开始后会对 `club_config` 做行锁。
   - 服务端以数据库最新值重新判断 `usedQuota/maxQuota` 和优惠名额，防止并发下超卖或错价成交。
   - 可用折扣判断条件：存在启用折扣、`discountRate` 不为空、`discountQuota > 0`、`useQuota < discountQuota`。
   - 折扣金额计算公式：`payAmount = admissionFee * (discountRate / 10)`，例如 `8` 表示 `8折`。
   - 若命中折扣，购买成功后会把 `club_config_discount.use_quota + 1`。
3. 会员购买规则：
   - 当前时间未超过 `validEnd` 时不可新购。
   - 超过 `validEnd` 后，若仍在宽限期内，是否允许新购由 `grace_action` 决定。
   - 超过宽限期后，是否允许新购由 `post_grace_action` 决定。
   - 会员状态为已失效时允许新购；已冻结、已停用时不允许新购。
4. 会员落库规则：
   - 新增 `club_member` 一条记录。
   - `accountNo` 同步写入当前账号编号，来源于 `account.account_no`。
   - `memberNo` 规则：`DGC + (100000 + memberId)`，例如 `DGC100001`。
   - `validStart`、`validEnd`、`graceTime` 写库时只保留到年月日，时分秒统一抹零。
   - `graceTime = validEnd + gracePeriodDays`；若无有效期或无宽限配置，则可能为空。
   - 当前实现不会在购买时写入 `pointsAmount`、`ticketNum` 等派发结果。
5. 订单和权益落库规则：
   - 新增 `club_order` 一条，订单类型为“新开卡”。
   - `club_order.order_no` 使用 `IdNoUtils.createOrderNo()` 生成。
   - `price` 记录原价，`discountPrice` 记录优惠金额（`price - actualAmount`），`actualAmount` 记录实付金额。
   - `club_order_benefit.order_no` 也使用 `IdNoUtils.createOrderNo()` 逐条生成。
   - 会为 `club_config_benefit` 中查到的每一条权益都新增一条 `club_order_benefit`，当前实现不按权益类型或状态过滤。
6. 安全密码说明：
   - 当前代码中购买接口保留了 `safetyPassword` 参数和非空校验。
   - 但实际安全密码校验代码已被注释，当前版本不会真的验密。

## 失败码
| 错误码 | 说明 |
|---|---|
| `10001` | 参数校验失败 |
| `10010` | 请求过于频繁，请稍后重试 |
| `10016` | 登录无效，请重新登录 |
| `10024` | 余额不足 |
| `10049` | Club不存在 |
| `10050` | 当前Club不可购买 |
| `10051` | 您已购买该Club |
| `10052` | 当前Club已售罄 |
| `10053` | Club价格已变化，请刷新后重试 |

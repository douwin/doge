# 分页查询积分记录

## 接口信息
| 项 | 内容 |
|---|---|
| 请求地址 | `/api/v1/club/member/points/list` |
| 请求方式 | `GET` |
| Token | 必填 |

## 业务说明
1. 分页查询指定会员的积分变动记录。
2. 当前接口要求显式传入 `memberId`，用于区分同账号下不同会员卡。

## 请求参数
| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| `clubConfigId` | String | 是 | Club配置ID |
| `memberId` | String | 是 | 会员ID |
| `pageNo` | Integer | 否 | 页码，默认 `1`，继承自 `PageReq` |
| `pageSize` | Integer | 否 | 每页数量，默认 `10`，继承自 `PageReq` |

## 请求示例
```http
GET /api/v1/club/member/points/list?clubConfigId=10001&memberId=20001&pageNo=1&pageSize=10
Language: zh_CN
Token: eyJhbGciOiJIUzI1NiJ9.demo
```

## 响应数据
`data` 为分页对象

`list` 元素结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | String | 积分记录ID |
| `sourceType` | Integer | 来源类型：`1=Club入会`，`2=Club续费`，`3=线下核销` |
| `sourceTypeName` | String | 来源类型名称 |
| `direction` | Integer | 方向：`1=收入`，`2=支出`，根据 `PointsSceneEnum.direction` 返回 |
| `beforePoints` | String | 变动前积分 |
| `points` | String | 本次变动积分 |
| `afterPoints` | String | 变动后积分 |
| `createTime` | String | 创建时间戳(毫秒) |

## 校验说明
1. `clubConfigId`、`memberId` 不能为空。
2. `clubConfigId`、`memberId` 必须可转为数值，否则返回参数校验失败。
3. Club 不存在时返回 `CLUB_NOT_EXISTS`。
4. `memberId` 对应会员不存在、会员不属于当前账号、或会员与 `clubConfigId` 不匹配时，返回 `CLUB_MEMBER_NOT_EXISTS`。

## 复杂业务说明
1. 数据来源于 `club_points_log`，查询条件为 `accountId + memberId`。
2. `direction` 取值来源于 `club_points_log.scene` 对应的 `PointsSceneEnum.direction`。
3. 排序规则为 `createTime desc, id desc`。
4. 当没有积分记录时，接口仍返回标准分页结构，`list=[]`，`totalSize=0`。

## 失败码
| 错误码 | 说明 |
|---|---|
| `10001` | 参数校验失败 |
| `10049` | Club不存在 |
| `10054` | Club会员不存在 |
